## 提取流程

### 第一步：侦察数据结构
在提取之前，先抽样分析（前10条或前5页）：
- 数据的实际结构是什么？
- 目标字段在哪里，格式如何？
- 有没有数据质量问题（缺失、不一致、格式混乱）？

向用户展示抽样结果，确认理解正确后再全量提取。

### 第二步：制定提取规则

```python
# 提取逻辑示意（在 scripts/ 中实现）
FIELD_MAP = {
    "output_field_name": "source_location_or_rule",
    # 例如：
    # "customer_name": "第一列",
    # "amount": "金额列（去掉货币符号，转为 float）",
    # "date": "日期列（统一转为 YYYY-MM-DD 格式）",
}

VALIDATION_RULES = {
    "必填字段": [...],
    "格式验证": {...},
}
```

### 第三步：全量提取

使用 `scripts/extract.py`（如果有）执行提取。执行过程中：
- 记录处理了多少条记录
- 记录跳过了多少条（以及原因）
- 记录遇到的数据质量问题

### 第四步：验证输出

```python
# 基本验证检查
assert len(results) > 0, "没有提取到任何数据"
assert all(r.get("id") for r in results), "存在缺少 ID 的记录"
# 抽样检查（输出前5条供人工确认）
```

## 输出格式

**JSON**（程序处理首选）：
```json
{
  "metadata": {
    "source": "文件路径",
    "extracted_at": "ISO 时间戳",
    "total_records": 42,
    "skipped_records": 2,
    "warnings": ["跳过第15行：缺少必填字段 amount"]
  },
  "records": [
    {"field1": "value1", "field2": "value2"}
  ]
}
```

**CSV**（人工查看/Excel 导入）：
- 标题行使用中文或英文（按用户偏好）
- 日期统一为 `YYYY-MM-DD`
- 数字不包含千分位分隔符

## 数据质量报告

每次提取完成后，输出简洁的数据质量摘要：
```
提取完成
- 总记录数：[N]
- 成功提取：[N]
- 跳过/异常：[N] 条（详见 extraction_warnings.log）
- 数据完整性：[X]%（所有必填字段均有值的记录占比）
```
