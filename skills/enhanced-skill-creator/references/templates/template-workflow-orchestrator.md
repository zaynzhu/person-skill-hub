## 错误恢复

如果工作流在某步骤中断：
1. 保存当前进度到 `workflow-state.json`
2. 告知用户中断的步骤和原因
3. 在下次运行时，从 `workflow-state.json` 读取状态，跳过已完成步骤

```json
// workflow-state.json 结构
{
  "started_at": "ISO 时间戳",
  "current_step": 2,
  "completed_steps": [1],
  "step_outputs": {
    "step_1": "输出文件路径或摘要"
  }
}
```
